Para instalar o sistema UniSis em um servidor Ubuntu 22.04, você precisará configurar o ambiente para Node.js (Backend e Frontend), PostgreSQL (Banco de Dados) e Python (Automação DANFE).

Abaixo está o passo a passo detalhado para realizar a instalação via terminal.

1. Atualizar o Sistema e Instalar Dependências Básicas
Primeiro, garanta que o sistema esteja atualizado e instale as ferramentas necessárias.

bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl build-essential python3 python3-pip nginx
2. Instalar Node.js (Versão 20 LTS)
Recomendo usar o Node.js 20 para garantir compatibilidade com o React 19 e Vite.

bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
3. Instalar e Configurar o PostgreSQL
O sistema utiliza PostgreSQL como banco de dados.

bash
# Instalar Postgres
sudo apt install -y postgresql postgresql-contrib
# Iniciar o serviço
sudo systemctl start postgresql
sudo systemctl enable postgresql
# Criar o banco de dados e usuário (Substitua 'sua_senha_aqui' pela senha do .env)
sudo -u postgres psql -c "CREATE DATABASE \"UniSis\";"
sudo -u postgres psql -c "CREATE USER postgres WITH PASSWORD 'Avg537901';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE \"UniSis\" TO postgres;"
4. Instalar Dependências do Python (DANFE)
O sistema usa a biblioteca pynfe para processar notas fiscais.

bash
pip3 install pynfe
5. Clonar o Repositório e Instalar Dependências da App
Vá para o diretório onde deseja instalar (ex: /var/www/unisis).

bash
sudo mkdir -p /var/www/unisis
sudo chown $USER:$USER /var/www/unisis
cd /var/www/unisis
# Clonar o código
git clone https://github.com/pfnh26/Unisis.git .
# Instalar dependências do Backend
cd backend
npm install
# Instalar dependências do Frontend
cd ../frontend
npm install
6. Configurar Variáveis de Ambiente
Você deve criar o arquivo 

.env
 dentro da pasta backend.

bash
nano /var/www/unisis/backend/.env
Cole o seguinte conteúdo (ajuste se necessário):

env
DB_USER=postgres
DB_HOST=localhost
DB_NAME=UniSis
DB_PASSWORD=Avg537901
DB_PORT=5432
JWT_SECRET=unisis_secret_Key_2024
PORT=5000
(Pressione Ctrl+O, Enter e Ctrl+X para salvar e sair).

7. Inicializar o Banco e Compilar o Frontend
Execute o script de criação de tabelas e gere os arquivos estáticos do React.

bash
# Inicializar tabelas (No backend)
cd /var/www/unisis/backend
node -e 'require("./db").initDb()'
# Gerar build do Frontend
cd /var/www/unisis/frontend
npm run build
8. Configurar o PM2 (Manter o Backend Rodando)
O PM2 garante que o servidor reinicie sozinho se cair.

bash
sudo npm install -g pm2
cd /var/www/unisis/backend
pm2 start index.js --name "unisis-backend"
pm2 save
pm2 startup
(Copie e cole o comando que o pm2 startup gerar no terminal).

9. Configurar Nginx (Servidor Web)
O Nginx servirá os arquivos do Frontend e redirecionará as chamadas de API para o Backend.

bash
sudo nano /etc/nginx/sites-available/unisis
Cole a configuração abaixo:

nginx
server {
    listen 80;
    server_name seu_dominio_ou_ip;
    root /var/www/unisis/frontend/dist;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
    location /api {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
    location /uploads {
        alias /var/www/unisis/backend/uploads;
    }
}
Ative a configuração e reinicie o Nginx:

bash
sudo ln -s /etc/nginx/sites-available/unisis /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
10. Permissões de Pastas
Garanta que o Node tenha permissão para salvar arquivos (uploads, certificados, etc).

bash
sudo chmod -R 777 /var/www/unisis/backend/uploads
sudo mkdir -p /var/www/unisis/backend/services/temp_docx
sudo chmod -R 777 /var/www/unisis/backend/services/temp_docx
Pronto! Agora você pode acessar o sistema digitando o IP do seu servidor Ubuntu no navegador. Se precisar atualizar o sistema no futuro, basta dar um git pull, rodar npm install e reiniciar o backend com pm2 restart unisis-backend.


git commit -m "v2.0"

git push

server

git pull origin main

cd backend
npm install --production
pm2 restart unisis-backend

cd /var/www/unisis/frontend
